# وێبسایتی پرێمیۆمی «تەفسیری نامی» (HTML/CSS/JS)

ئەم پڕۆژەیە وێب-ئەپێکی ستاتیکە (بێ بەک-ئێند) بۆ خوێندنەوەی PDF ـی «تەفسیری نامی»، بە دیزاینی زۆر جوان و تایبەتمەندیەکان:

- خوێندنەوەی PDF بە PDF.js
- پێڕست/Outline (ئەگەر لە PDF ـدا هەبێت)
- گەڕان (Search) + بنیاتنانی ئیندێکس بۆ خێراتر
- نیشانەکردن (Bookmarks)
- تێبینی (Notes) بۆ هەر پەڕەیەک
- ڕووناک/تاریک/سێپیا
- پێوانەی پەڕە (Fit width / Fit page) + زووم
- پاشەکەوتی دوایین پەڕە و ڕێکخستنەکان
- PWA + Offline cache (بەکارهێنانی بێ ئینتەرنێت دوای یەک جار بارکردن)

## چۆن کار بکات؟
### 1) بۆ مۆبایل (iPhone)
- ئەم پڕۆژەیە پێویستی بە HTTP/HTTPS هەیە (فایل-سیستەم file:// زۆرجار fetch ناکات).
- باشترین ڕێگا:
  1) بڵاوکردنەوە لە GitHub Pages / Netlify / Vercel / Cloudflare Pages
  2) دواتر لە Safari → Share → “Add to Home Screen”

### 2) بۆ کۆمپیوتەر
- بە سادەیی لە فۆڵدەری پڕۆژە:
  - `python -m http.server 8080`
  - دواتر بکە `http://localhost:8080`

## PDF
فایلەکە لە `assets/tafsir-nami.pdf` ـدایە. دەتوانیت ناوی بگۆڕیت، بەڵام لە `app.js` ـدا `PDF_URL` ـیش دەبێت بگۆڕیت.

> تێبینی: ئەگەر فایلەکە زۆر گەورە بێت و ناتوانیت لە GitHub Pages دایبنێیت، پێویستە PDF لە هۆستێکی تر دابنێیت (Cloudflare R2, Google Drive direct, Git LFS, ...)، و لە `PDF_URL` ناونیشانەکەی دابنێیت.
